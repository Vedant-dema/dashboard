## FEATURE-081: Global Real-World Address Autocomplete ‚Äî GlobalAddressSearch
**Date:** 2026-03-30
**Author/Agent:** Cursor AI
**Status:** Added

### What Was Added
A global address search field added to the "Kunde / Standort" section of the New Offer (Neues Angebot) form. As the user types any address fragment (street, city, postcode, country name), real-world suggestions are fetched live from the Photon/OpenStreetMap API covering all countries worldwide. Selecting a result instantly auto-fills PLZ, Ort, and Land. The Land dropdown was also expanded from 5 to 47 countries.

### Where It Was Added
- `frontend/src/components/GlobalAddressSearch.tsx` ‚Äî New live-fetch address autocomplete component (created)
- `frontend/src/components/NewAngebotModal.tsx` ‚Äî Added GlobalAddressSearch field; expanded LAND_OPTIONS to 47 countries; replaced CustomerAddressPicker with plain input for Firmenname; added `handleAddressSelect` callback; added `useLanguage` hook; shows selected address preview beneath the search field
- `frontend/src/pages/AngebotePage.tsx` ‚Äî Removed unused kundenStore import and customerCandidates (no longer needed)
- `frontend/src/contexts/LanguageContext.tsx` ‚Äî 7 new i18n keys added (DE / EN / FR)
- `frontend/src/components/CustomerAddressPicker.tsx` ‚Äî Kept (still useful for other forms) but decoupled from Angebot modal

### What It Does (Technical)
1. User types ‚â• 2 characters in the address search field.
2. A 400ms debounce fires, then a `fetch` to `https://photon.komoot.io/api/?q={query}&limit=8&lang={lang}`.
3. `parsePhoton()` maps Photon GeoJSON features to `{label, strasse, plz, ort, land_code, land_name}`.
4. Dropdown opens with up to 8 results; each item shows a country flag emoji, primary address line, secondary PLZ+Ort line, and country code badge.
5. Arrow-key + Enter keyboard navigation; Escape to close; click outside to close.
6. `AbortController` cancels in-flight requests on each new keystroke.
7. On selection: `handleAddressSelect` copies `plz`, `ort`, `land_code` into the form state.
8. A compact preview line beneath the search field confirms what was filled.
9. If the geocoded country code is not in LAND_OPTIONS, a fallback `<option>` is rendered dynamically so the value is not lost.

|-------|------|----------|-------------|

| className | string | no | CSS classes for the input |

## FEATURE-080: Global Customer Address Autofill ‚Äî CustomerAddressPicker
**Date:** 2026-03-30
**Author/Agent:** Cursor AI
**Status:** Added

### What Was Added
When a user types a company name (Firmenname) in the New Offer (Neues Angebot) form, a smart dropdown appears showing matching customers from the customer database. Selecting a customer instantly auto-fills all address fields ‚Äî PLZ, Ort, and Land ‚Äî saving manual entry and reducing errors from mistyped addresses.

### Where It Was Added
- `frontend/src/components/CustomerAddressPicker.tsx` ‚Äî New reusable combobox component (created)
- `frontend/src/components/NewAngebotModal.tsx` ‚Äî Firmenname field replaced with CustomerAddressPicker; `handleCustomerSelect` callback added; `customerCandidates` prop added
- `frontend/src/pages/AngebotePage.tsx` ‚Äî Loads kundenDb, derives `customerCandidates`, passes to NewAngebotModal
- `frontend/src/contexts/LanguageContext.tsx` ‚Äî 4 new i18n keys added (DE / EN / FR)

### What It Does (Technical)
1. `AngebotePage` loads `kundenDb` from localStorage via `loadKundenDb()` on mount.
2. It derives `customerCandidates` (a useMemo) ‚Äî a slim array of `{firmenname, strasse, plz, ort, land_code}` for all non-deleted customers.
3. Candidates are passed as a prop to `NewAngebotModal`.
4. `CustomerAddressPicker` renders a text input + custom dropdown. As the user types, it filters candidates by `firmenname` (case-insensitive) and shows up to 8 matches.
5. Each dropdown item shows the company name (with the typed query highlighted) and a secondary line with street + PLZ + Ort + country badge.
6. Arrow-key and Enter keyboard navigation is fully supported; Escape closes the dropdown.
7. On click/Enter selection, `onSelect(candidate)` fires and `NewAngebotModal.handleCustomerSelect` copies `firmenname`, `plz`, `ort`, `land_code` into the form state.
8. The user can still edit the auto-filled fields manually after selection.

## FEATURE-074: Duplicate Customer Prevention ‚Äî Name-Match Warning on Create
**Date:** 2026-03-30  
**Author/Agent:** Cursor AI  
**Status:** Added  

### What Was Added
When a user attempts to create a new customer whose company name exactly matches (case-insensitive) an existing customer, a blocking confirmation overlay appears inside the modal listing all matching customers (KU-NR + company name). The user can either cancel and change the name, or explicitly choose "Save anyway" to proceed. Edit mode is not affected ‚Äî only the "Add customer" flow has this guard.

### Where It Was Added
- `frontend/src/components/NewCustomerModal.tsx` ‚Äî added `duplicateCheck` optional prop, `showDuplicateWarning` and `duplicateMatches` state, guard logic in `handleSave`, `handleDuplicateSaveAnyway` handler, and the amber-styled confirmation overlay

- `frontend/src/contexts/LanguageContext.tsx` ‚Äî added i18n keys in DE, EN, FR

### What It Does (Technical)
1. User fills the company name field and clicks "Save" in create mode.
2. `handleSave` calls `duplicateCheck(form.firmenname.trim())` if the prop is provided.
3. The check in `CustomersPage` normalises both names to lowercase and compares for exact equality, returning any matching customers.
4. If matches exist, `showDuplicateWarning` is set to `true` and the amber overlay renders inside the modal, listing each duplicate with KU-NR and name.
5. "Cancel" dismisses the overlay so the user can edit the name.
6. "Save anyway" calls `handleDuplicateSaveAnyway`, which clears the warning and continues the normal save flow (including the scanned-attachment prompt if applicable).
