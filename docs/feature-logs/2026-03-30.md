## FEATURE-072: Customer Risk Analysis — Smart Document Expiry Tracker
**Date:** 2026-03-30
**Author/Agent:** Cursor AI
**Status:** Added

### What Was Added
A per-customer "Risikoanalyse" (Risk Analysis) panel inside the customer detail drawer. It stores expiry / valid-until dates for 5 compliance documents (Registerauszug, Wirtschaftsberichts-Ermächtigung, Ausweis-Kopie + Wirt.-Ber., Ausweis gültig bis, Ausweis-Kopie Abholer) plus two boolean flags (Allg. Dok.-Bogen, Verst. Dok.-Bogen) and a Bearbeiter text field. Color-coded status badges (Expired/Critical/Warning/Valid/Missing) display instantly per document. An alert banner pops up at the top of the panel when any document is expired or near expiry. Customers with expiry issues also show a warning triangle icon in the customer list.

### Where It Was Added
- `frontend/src/types/kunden.ts` — Added `KundenRisikoanalyse` interface; updated `KundenDbState` with `risikoanalysen[]` and `nextRisikoanalyseId`
- `frontend/src/store/kundenStore.ts` — Added `getRisikoanalyseForKunde`, `upsertRisikoanalyse`, `getExpiryStatus`, `daysUntilExpiry`, `hasRisikoAlert`; updated `normalizeUnterlagen` and `seedDb` for the new fields
- `frontend/src/pages/CustomersPage.tsx` — Added Risikoanalyse section UI with alert banner, read-only table, inline edit form; warning badge on list rows; imported new store functions
- `frontend/src/contexts/LanguageContext.tsx` — Added 27 new i18n keys (de/en/fr) under the `risk*` prefix

### What It Does (Technical)
1. `KundenRisikoanalyse` stored in `db.risikoanalysen[]` (one record per `kunden_id`) in localStorage and the shared demo API blob.
2. `getExpiryStatus(isoDate)` classifies a date as `expired | critical | warning | ok | missing` relative to today.
3. `hasRisikoAlert(risk)` returns `true` if any date field is `expired | critical | warning` — used to pre-compute `alertKundenIds` Set rendered as an amber warning triangle in the customer list.
4. When a customer row is opened, the Risikoanalyse section reads the current record; if any docs are problematic, a red banner lists the counts.
5. "Edit dates" button opens an inline form with date inputs and checkboxes; on Save, `upsertRisikoanalyse` writes the record and persists.
