**Author/Agent:** Cursor AI
**Status:** Added

### What Was Added
Users can increase or decrease text size from **Settings ‚Üí Display** using a slider and ¬± controls. The choice is stored with other app settings and applied across the whole dashboard immediately (including after reload).

### Where It Was Added

4. The same index is written to `localStorage` under `dema-app-settings.fontSizeStepIndex` with the existing settings blob.

## FEATURE-110: Settings text size ‚Äî typography-only (no full-page zoom)
**Date:** 2026-03-31
**Author/Agent:** Cursor AI
**Status:** Extended (FEATURE-109)

### What Was Added
The text-size control no longer changes root `html` font-size (which scaled padding, gaps, and icons like browser zoom). It now sets `--dema-font-scale` on `document.documentElement`, and Tailwind‚Äôs standard `text-*` utilities use `calc(... * var(--dema-font-scale))` so **only typography** from those tokens grows or shrinks. Steps were narrowed to a subtler band (roughly 94%‚Äì114% of default text). Legacy inline `font-size` on `html` is cleared when applying.

### Where It Was Added
- `frontend/tailwind.config.js` ‚Äî `demaScaledFontSize()` + `extend.fontSize` overrides for `xs` ‚Ä¶ `9xl`
- `frontend/src/index.css` ‚Äî default `html { --dema-font-scale: 1 }`
- `frontend/src/common/utils/appFontScale.ts` ‚Äî `FONT_SIZE_STEP_MULTIPLIERS`, `setProperty('--dema-font-scale')`, `removeProperty('font-size')`, `fontSizeStepLabelPercent`
- `frontend/src/pages/SettingsPage.tsx` ‚Äî slider bounds / labels; help uses `text-xs`
- `frontend/src/contexts/LanguageContext.tsx` ‚Äî updated `settingsFontSizeHelp` (de / en / fr)

### What It Does (Technical)
1. `applyFontSizeStepIndex` writes `--dema-font-scale` to the root element and removes any previous `font-size` style.
2. Tailwind-generated rules for default font sizes multiply both `font-size` and bundled `line-height` (where rem-based) by that variable; unitless line-heights (`1`) on display sizes stay unitless.
3. Spacing (`p-*`, `gap-*`, `h-*`, `w-*` for layout) remains tied to normal root rem and does not scale with the control.

## FEATURE-116: Optional live UI translation API (Google / LibreTranslate)
**Date:** 2026-03-31
**Author/Agent:** Cursor AI
**Status:** Added

### What Was Added
When **`VITE_ENABLE_LIVE_UI_TRANSLATION=true`** (frontend build) and the backend has **`GOOGLE_TRANSLATE_API_KEY`** or **`LIBRETRANSLATE_URL`**, the nine secondary dashboard languages load **machine translations from the English message catalog** in batches and overlay them in **`t()`** as each batch returns‚Äîsimilar in spirit to ‚Äútranslate the whole UI,‚Äù but without exposing API keys in the browser. **Deutsch, English, Fran√ßais** stay fully static (hand-authored). Settings shows a short **amber notice** when live translation is enabled at build time.

### Where It Was Added
- `backend/main.py` ‚Äî `GET /api/v1/ui-translate/status`, `POST /api/v1/ui-translate/batch`; Google v2 + LibreTranslate client helpers
- `backend/.env.example` ‚Äî documented env vars
- `frontend/src/services/uiTranslateApi.ts` ‚Äî status + batch client
- `frontend/src/contexts/LanguageContext.tsx` ‚Äî batch hydration `useEffect`, `t()` overlay; i18n keys `settingsLiveTranslationBanner` (de/en/fr)
- `frontend/src/pages/SettingsPage.tsx` ‚Äî banner when `VITE_ENABLE_LIVE_UI_TRANSLATION`
- `frontend/.env.example`, `frontend/src/vite-env.d.ts` ‚Äî `VITE_ENABLE_LIVE_UI_TRANSLATION`

### What It Does (Technical)
1. On `language` change into `es|it|pt|tr|ru|hi|ar|zh|ja`, the provider clears prior overlays and calls `/ui-translate/status`; if enabled, it chunks `Object.entries(MESSAGES.en)` (48 strings per request), posts to `/ui-translate/batch`, merges `translations` by key, and calls `setMtByKey` after each chunk so the UI updates progressively.
2. `t(key)` returns `mtByKey[key]` when present and non-empty; otherwise the existing static merge (`MESSAGES[language]` ‚Üí `MESSAGES.en` ‚Üí fallback).
3. Google path sends one HTTP request per batch; LibreTranslate path calls `/translate` once per string (compatible with common server implementations).

### Data It Accepts / Emits
| Field | Type | Required | Description |
|----|---|----|----|
| `texts` | string[] | yes | Up to 80 segments per batch; total chars capped server-side (~14k) |
| `source` | string | no | Default `en` |
| `target` | string | yes | e.g. `tr`, `zh-CN` |

### Database
N/A

### API Endpoints (if applicable)
| Method | Path | Auth | Request Body | Response |
|-----|---|---|----|----|
| GET | /api/v1/ui-translate/status | No | ‚Äî | `{ enabled, provider }` |
