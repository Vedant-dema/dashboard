## FEATURE-109: Settings ŌĆö Global text size (font scale)
**Date:** 2026-03-31
**Author/Agent:** Cursor AI
**Status:** Added

### What Was Added
Users can increase or decrease text size from **Settings ŌåÆ Display** using a slider and ┬▒ controls. The choice is stored with other app settings and applied across the whole dashboard immediately (including after reload).

### Where It Was Added
- `frontend/src/common/utils/appFontScale.ts` ŌĆö step list (88%ŌĆō125%), clamp/apply/read/hydrate helpers

- `frontend/src/contexts/LanguageContext.tsx` ŌĆö i18n keys (de / en / fr)

### What It Does (Technical)

3. Settings keeps `fontSizeStepIndex` in React state; a `useEffect` calls `applyFontSizeStepIndex` on every change for instant updates.
4. The same index is written to `localStorage` under `dema-app-settings.fontSizeStepIndex` with the existing settings blob.

## FEATURE-110: Settings text size ŌĆö typography-only (no full-page zoom)
**Date:** 2026-03-31
**Author/Agent:** Cursor AI
**Status:** Extended (FEATURE-109)

### What Was Added
The text-size control no longer changes root `html` font-size (which scaled padding, gaps, and icons like browser zoom). It now sets `--dema-font-scale` on `document.documentElement`, and TailwindŌĆÖs standard `text-*` utilities use `calc(... * var(--dema-font-scale))` so **only typography** from those tokens grows or shrinks. Steps were narrowed to a subtler band (roughly 94%ŌĆō114% of default text). Legacy inline `font-size` on `html` is cleared when applying.

### Where It Was Added
- `frontend/tailwind.config.js` ŌĆö `demaScaledFontSize()` + `extend.fontSize` overrides for `xs` ŌĆ” `9xl`
- `frontend/src/index.css` ŌĆö default `html { --dema-font-scale: 1 }`

- `frontend/src/pages/SettingsPage.tsx` ŌĆö slider bounds / labels; help uses `text-xs`
- `frontend/src/contexts/LanguageContext.tsx` ŌĆö updated `settingsFontSizeHelp` (de / en / fr)

### What It Does (Technical)
1. `applyFontSizeStepIndex` writes `--dema-font-scale` to the root element and removes any previous `font-size` style.
2. Tailwind-generated rules for default font sizes multiply both `font-size` and bundled `line-height` (where rem-based) by that variable; unitless line-heights (`1`) on display sizes stay unitless.
3. Spacing (`p-*`, `gap-*`, `h-*`, `w-*` for layout) remains tied to normal root rem and does not scale with the control.
