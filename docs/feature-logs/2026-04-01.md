## FEATURE-134: Bulgarian VIES address — `Gr.` / `гр.` city suffix after comma
**Date:** 2026-04-01
**Author/Agent:** Cursor AI
**Status:** Modified

### What Was Added
VIES addresses such as `… Obl.Plovdiv, Gr.Plovdiv` (Bulgarian *Grad* = city) now split into street and city: the part after the last comma is recognised when it starts with `Gr.` or Cyrillic `гр.`, and the city name is stored without that prefix. Multiline addresses whose last line is only `Gr.City` are handled the same way.

### Where It Was Added
- `frontend/src/components/NewCustomerModal.tsx` — `tryGradAbbrevTailCity`, `parseViesSingleLineAddress`, `parseViesMultilineAddressBody`

### What It Does (Technical)
1. After a comma/semicolon split, if `tryPostalAndCity` fails on the right segment, `tryGradAbbrevTailCity` tests `^Gr\.?\s*(.+)$` and `^гр\.?\s*(.+)$` (Unicode flag).
2. Multiline: if the last non-country line matches that pattern, preceding lines become `strasse` and the captured token becomes `ort`.
3. Numeric PLZ is unchanged: if the registry string contains no `NNNN City` pattern, `plz` can remain empty (member-state dependent).

### Data It Accepts / Emits
| Field | Type | Required | Description |
|----|---|----|----|
| address (VIES) | string | No | May contain `, Gr.City` or `, гр. …` tail |

## FEATURE-135: Address transliteration (BG Cyrillic) — no bogus “Jk” / title-case mangling
**Date:** 2026-04-01
**Author/Agent:** Cursor AI
**Status:** Modified

### What Was Added
Address fields use a dedicated pipeline: Bulgarian Cyrillic abbreviations (`жк`, `обл.`, `гр.`, `бл.`, `вх.`, `ет.`, `ап.`, etc.) map to stable Latin (`Zh.K.`, `obl.`, `gr.`, …) before generic transliteration, then scientific BG letter mapping so `ЖК` is not rendered as `Jk`. VIES-derived street/city/plz no longer run through title-case (which broke `bl.70`, `Zh.K.`, etc.). VAT name/address blocks use `translate="no"` / `notranslate` to reduce browser auto-translate turning `obl.`/`gr.` into words like “Region”/“Stadt”. Backend skips unidecode on `BG` VIES display text like `DE`, preserving registry Cyrillic until the frontend normalizes.

### Where It Was Added
- `frontend/src/common/utils/addressTransliterate.ts` — new
- `frontend/src/common/utils/addressLatinNormalize.ts` — uses `transliterateAddressToAscii`
- `frontend/src/components/NewCustomerModal.tsx` — VIES UI + parsing merge (restored multiline/PLZ/nested raw + Gr. handling), `looksAdministrativeAreaText` uses address transliteration
- `backend/main.py` — `_normalize_vies_display_text` preserves `BG` like `DE`

### What It Does (Technical)
1. `expandCyrillicAddressAbbreviations` then `mapBulgarianCyrillicToLatin`, then existing `transliterateToAscii` for any remaining scripts.
2. `normalizeLatinAddressLine` calls the address-specific pipeline.
3. VIES parse outputs use `normalizeLatinAddressLine` only (no `toTitleCaseLatin`).
4. `normalizeVatAddressForUi` uses `transliterateAddressToAsciiMultiline` without per-line title case.
