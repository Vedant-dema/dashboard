## FEATURE-145: New/Edit customer modal — close guard when form differs from open baseline
**Date:** 2026-04-07
**Author/Agent:** Cursor AI
**Status:** Added

### What Was Added
Closing the customer modal (backdrop, **X**, or **Cancel**) prompts for confirmation when the serialized form state **differs from the baseline** captured on open — **create and edit**. **Keep editing** dismisses the dialog; **Discard** closes the modal. **Escape** dismisses the confirm overlay. If the confirm is already open, another backdrop close dismisses the confirm only.

### Where It Was Added
- `frontend/src/components/NewCustomerModal.tsx` — `showDiscardConfirm`, `requestClose`, `performCloseDiscard`, overlay `role="alertdialog"`
- `frontend/src/contexts/LanguageContext.tsx` — `newCustomerDiscardTitle`, `newCustomerDiscardMsg`, `newCustomerDiscardConfirm`, `newCustomerDiscardStay` (de, en, fr)

### What It Does (Technical)
1. `draftDiffersFromOpen` compares `serializeFormStateForDirtyBaseline(form)` to `dirtyBaselineRef` (set when the modal transitions closed → open).
2. Footer “unsaved changes” banner remains **edit-only** via `isFormDirty = isEditMode && draftDiffersFromOpen`.
3. Discard overlay uses `z-[30]` above duplicate warning (`z-20`).

### i18n Keys Added
`newCustomerDiscardTitle`, `newCustomerDiscardMsg`, `newCustomerDiscardConfirm`, `newCustomerDiscardStay`

### Notes / Known Limitations
None beyond existing dirty-serialization scope. `npm run build` verified.

## FEATURE-146: Customer modal productivity — keyboard, deep link, copy, validation jump, dirty banner
**Date:** 2026-04-07
**Author/Agent:** Cursor AI
**Status:** Added

### What Was Added
The new/edit customer modal is faster to use: **Ctrl/Cmd+S** saves, **Escape** closes (with discard confirm when dirty), **Alt+1…n** switches tabs in order. The **Customer 360** summary supports **jump to** company, primary address, and VAT fields on the Kunde & Adresse tab; **one-click copy** for customer number, VAT string, and formatted address. A short **“Unsaved changes”** banner appears whenever the draft differs from the open baseline (create and edit). **Save validation** shows a compact footer line with “+N more” and focuses the first invalid field. The customers list URL supports **`?customer=KUNDENNR`** on the hash route so bookmarks and reload reopen the same row; closing the modal clears or updates the query. Copy-flash timers are cleared on unmount.

### Where It Was Added
- `frontend/src/App.tsx` — `getRouteFromHash` ignores `?` query when resolving route
- `frontend/src/components/Sidebar.tsx` — same hash path parsing if duplicated
- `frontend/src/pages/CustomersPage.tsx` — `readCustomerParamFromHash`, `writeCustomerHashPreservePath`, `hashchange` + ref guard to open/strip `customer` param
- `frontend/src/components/NewCustomerModal.tsx` — focus IDs, `focusModalCustomerField`, `handleSaveRef`, keyboard `useEffect`, summary jump/copy, dirty banner from `draftDiffersFromOpen`, validation aggregation + tab switch + focus, `aria-describedby` shortcuts hint, copy timer cleanup
- `frontend/src/contexts/LanguageContext.tsx` — de/en/fr keys for copy, shortcuts hint, validation more, invalid email, ARIA/go-to labels

### What It Does (Technical)
1. Hash routing: path segment before `?` maps to app route; query holds `customer=<kunden_nr>`.
2. Customers page syncs selection with hash on load and `hashchange`; invalid `customer` values are stripped; same-Kunde re-open skipped via ref to avoid resetting on unrelated `db` updates.
3. Modal: global keydown when open (respecting discard/duplicate overlays); save ref always points at latest `handleSave`.
4. `handleSave` builds ordered validation issues (required company name; primary contact email format when non-empty); sets `footerMessage` with `moreCount`; switches tab and focuses first issue field via stable DOM ids.
5. Summary strip: buttons call tab switch + `activeAdresseIdx` 0 + `focusModalCustomerField`; copy uses `navigator.clipboard.writeText` with brief ok/err flash.
6. `isFormDirty` uses full draft-vs-baseline comparison for both create and edit.

### Data It Accepts / Emits
| Field | Type | Required | Description |
|----|---|----|----|
| URL `customer` | string (Kunden-Nr.) | No | Deep link target on `#/sales/kunden?customer=…` |
| Clipboard | text | No | Ku-Nr., VAT line, address line on user action |

### Database

- **Tables Affected:** N/A
- **Schema Changes:** None
- **Key Queries:** None
