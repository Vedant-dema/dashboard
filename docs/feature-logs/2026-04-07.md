## FEATURE-136: Backend — JSON on unhandled errors; VIES requester misconfig as 503; CORS regex
**Date:** 2026-04-07
**Author/Agent:** Cursor AI
**Status:** Modified

### What Was Added
Unhandled Python exceptions now return a **JSON body** (`detail` + `error_type`) instead of risking an empty 500. **Half-set** `VIES_REQUESTER_CC` / `VIES_REQUESTER_VAT` env vars raise **503** with a clear `detail` (previously 500). Optional **`CORS_ORIGIN_REGEX`** is supported next to **`CORS_ORIGINS`**. Startup logs CORS settings. VAT check empty-body hints in the UI mention requester env and CORS regex for 503.

### Where It Was Added
- `backend/main.py` — `_cors_origin_regex`, lifespan log, `CORSMiddleware` `allow_origin_regex`, `_unhandled_exception_handler`, `_default_requester_from_env` status/detail text
- `frontend/src/components/NewCustomerModal.tsx` — `runVatCheck` empty-body messages for 503 and generic 5xx
