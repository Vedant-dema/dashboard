## FEATURE-142: VAT check — hard asyncio deadline + structured detail parsing in UI
**Date:** 2026-04-08  
**Author/Agent:** Cursor AI  
**Status:** Added  

### What Was Added
**Backend:** `POST /api/v1/vat/check` is wrapped in `asyncio.wait_for` with timeout `VIES_CHECK_ENDPOINT_MAX_TOTAL_SEC` (min 2 s) **+ 1.5 s**. If the handler overruns, the API returns **HTTP 504** with JSON `detail: { code: "vat_check_deadline_exceeded", message: "..." }` so clients get a structured error instead of relying on the gateway to time out with an empty body. **Frontend:** VAT error formatting now treats FastAPI **`detail` objects** (`{ code, message }`) and **arrays** under `detail` so the modal shows real messages instead of falling through to generic “no detail” copy.

### Where It Was Added
- `backend/main.py` — `_check_vat_impl`, `check_vat` deadline wrapper; module docstring env note
- `backend/.env.example` — comment on hard deadline vs proxy
- `backend/app/tests/test_vat_error_contract.py` — `test_vat_check_hard_deadline_returns_504_json`
- `frontend/src/components/NewCustomerModal.tsx` — `vatErrorBodyHasDetail`, `formatVatCheckDetail`
- `docs/FEATURE-LOG.md`, `docs/feature-logs/2026-04-08.md` — this entry

### What It Does (Technical)
1. `check_vat` reads endpoint budget from env, computes `hard_cap_sec = budget + 1.5`, awaits `_check_vat_impl` under `wait_for`.
2. On `asyncio.TimeoutError`, log warning and raise `HTTPException(504, detail=dict)`.
3. UI: if `detail` is an object, show `[code] message` when present; recurse for array-shaped `detail`.
