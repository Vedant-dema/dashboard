## FEATURE-137: VAT check error copy — empty HTTP body & CORS docs
**Date:** 2026-04-08  
**Author/Agent:** Cursor AI

2. If JSON parses but has no `detail` / `message` / `error` payload, show 405-specific vs generic empty-body guidance.
3. `formatVatCheckDetail` fallback uses `t("vatCheckErrorNoServerDetail", ...)`.

## FEATURE-138: VIES response — title case for postal address
**Date:** 2026-04-08  
**Author/Agent:** Cursor AI  
**Status:** Modified  

### What Was Added
The VAT check API now formats the **postal address** field (`address`, `address_normalized`) in **title case** per line after existing country-aware display normalization. **Company / trader name** is unchanged (legal string). `address_original` and `address_search` stay derived from registry text as before. `normalization_version` is set to `vies_norm_v2_address_title`.

### Where It Was Added
- `backend/main.py` — `_title_case_vies_address`, applied in `_map_vies_check_response` before the “Not provided by VIES” placeholder.

### What It Does (Technical)
1. After `_normalize_vies_display_text` (and optional fallback address), each non-empty line of the address is passed through `str.strip().title()`.
2. Multiline addresses keep line breaks; blank lines are preserved in the join.
3. Placeholder text is not title-cased.

### Data It Accepts / Emits
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| address | string | No | Display address; title case per line |
| address_normalized | string | No | Same as display policy |
| normalization_version | string | No | `vies_norm_v2_address_title` |
