## FEATURE-144: Shared customers load — visible errors and env trim
**Date:** 2026-04-09  
**Author/Agent:** Cursor AI  
**Status:** Modified  

### What Was Added
When **API mode** is enabled but the first **GET `/api/v1/demo/customers-db`** fails (CORS, 401, network, etc.), the Customers page now shows an **amber banner** with a clear explanation and optional **technical message**, instead of failing **silently** and leaving users thinking only local storage applies. **`VITE_CUSTOMERS_SOURCE`** and **`VITE_API_BASE_URL`** are **trimmed** at build time so accidental spaces in Vercel env values no longer disable API mode.

### Where It Was Added
- `frontend/src/pages/CustomersPage.tsx` — initial shared-load `catch` sets `syncNotice` with i18n
- `frontend/src/store/kundenStore.ts` — `.trim()` on customer source and API base URL
- `frontend/src/contexts/LanguageContext.tsx` — `customersSharedLoadFailed*`, `customersSharedLoadDetail` in de/en/fr
- `docs/FEATURE-LOG.md`, this file — FEATURE-144

### What It Does (Technical)
1. `useEffect` for shared load: on error, builds a notice from `t("customersSharedLoadFailed")`, `t("customersSharedLoadFailedHint")`, and optional `t("customersSharedLoadDetail")` with `{detail}` from `Error.message`.
2. `API_MODE` and `API_BASE` use trimmed env strings before comparison / URL construction.

|-------|------|----------|-------------|
| N/A | N/A | N/A | UI copy and env parsing only |

- **Tables Affected:** N/A
- **Schema Changes:** None
- **Key Queries:** None

### API Endpoints (if applicable)

### State / Store (if applicable)

- **Persisted:** No

### i18n Keys Added
`customersSharedLoadFailed`, `customersSharedLoadFailedHint`, `customersSharedLoadDetail` (de, en, fr).

## FEATURE-145: Purchase Kalender — Timetable UI (call planner prototype)
**Date:** 2026-04-09  
**Author/Agent:** Cursor AI  
**Status:** Added  

### What Was Added
A dedicated **Purchase Kalender** screen at `#/purchase/kalender` (alias `#/purchase/timetable`) replacing the previous fallback to the home dashboard. **Modern UI** (gradient hero, glass table, rose/amber palette, personalized greeting from session user) mirroring the legacy **Termin-Kalender** workflow: buyer/processor filter, daily overview table, quick filters, date range + mini calendar, localStorage-backed rows with seed data, **new call slot** modal, **trucks available** follow-up modal appending to notes, **insert five drafts**, print list, toggle done, `tel:` links, and **offers-only (KA)** checkbox.

### Where It Was Added
- `frontend/src/App.tsx` — `TIMETABLE_ROUTES`, render `TimetablePage`
- `frontend/src/pages/timetable/TimetablePage.tsx` — main page UI and logic
- `frontend/src/pages/timetable/TimetableMiniCalendar.tsx` — month grid, locale-aware weekdays
- `frontend/src/pages/timetable/index.ts` — barrel export
- `frontend/src/types/timetable.ts` — `TimetableEntry`, filter mode types
- `frontend/src/store/timetableStore.ts` — `localStorage` load/save, seed rows, CRUD helpers
- `frontend/src/contexts/LanguageContext.tsx` — `timetable*` keys (de, en, fr)
- `docs/FEATURE-LOG.md`, this file — FEATURE-145

### What It Does (Technical)
1. On load, `loadTimetableDb()` reads `dema-purchase-timetable-v1` or seeds five demo rows for **today** (ISO date).
2. `filteredRows` applies date range, optional processor `vp`, optional `KA`-only filter, and filter mode (`day`, unfinished, other buyer, done without follow-up, parked).
3. Mutations call `saveTimetableDb` via `useEffect` on `db` changes.
4. Mini calendar updates selected day and syncs range to that day when picking a cell.
