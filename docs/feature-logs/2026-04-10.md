## FEATURE-152: DEMA canvas — shared home + customers ambient; modal backdrop vibe
**Date:** 2026-04-10  
**Author/Agent:** Cursor AI  
**Status:** Extended  

### What Was Added
A unified **DEMA canvas** ambient layer (animated mesh base, four soft orbs, sweeping shine, drifting grid, noise, vignette) now appears behind both the **dynamic dashboard (home)** and the **customers** list page. The customer create/edit **modal overlay** uses a **cinematic** multi-gradient backdrop class with a gentle brightness pulse and slightly stronger blur. Legacy **`customers-vibe-*`** page background utilities were replaced by **`dema-canvas-*`**; **`@keyframes`** and **`prefers-reduced-motion`** were aligned so motion-sensitive users get static backgrounds.

### Where It Was Added
- `frontend/src/index.css` — `dema-canvas-*` layers (already present), new keyframes (`dema-canvas-base-shift`, orb/shine/grid, `dema-modal-backdrop-flow`), reduced-motion overrides for canvas + modal backdrop; removed obsolete `customers-vibe-*` keyframes
- `frontend/src/pages/CustomersPage.tsx` — root/ambient/content classes switched to `dema-canvas-*` (+ vignette + shine + fourth orb)
- `frontend/src/pages/DynamicDashboard.tsx` — same canvas wrapper around dashboard content
- `frontend/src/components/NewCustomerModal.tsx` — overlay uses `dema-modal-backdrop-vibe` + `backdrop-blur-[3px]`

### What It Does (Technical)
1. **`dema-canvas-root`** isolates stacking; **`dema-canvas-ambient`** is `absolute inset-0` with non-interactive children.
2. Base gradient layers animate **`background-position`** on a long alternate loop; orbs use independent **`transform`** / opacity loops; shine and grid use sweep/drift animations.
3. **`dema-modal-backdrop-vibe`** layers radial gradients over a dark base and animates **`filter: brightness/saturate`** for a subtle living backdrop behind the dialog.
4. **`prefers-reduced-motion: reduce`** disables those animations (and clears modal filter animation).
