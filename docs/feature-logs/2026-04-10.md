## FEATURE-175: Kundenkontakt header — customer no. & industry from Kundenstamm
**Date:** 2026-04-10  
**Author/Agent:** Cursor AI  
**Status:** Modified  

### What Was Added
**Customer number** and **industry (Branche)** in the contact drawer header are **read-only** and sourced from the **Customers** master DB (`kundenStore`): new **`resolveKundeForTimetableRow`** matches first by `contact_profile.customer_number` === `kunden_nr`, else by normalized **company name** vs `firmenname` (non-deleted rows). The header shows a **teal-tinted CRM snapshot** (Building icon, two columns, unlinked hint). **Ka / Arte** remain editable in the header.

### Where It Was Added
- `frontend/src/store/kundenStore.ts` — `resolveKundeForTimetableRow`, `normalizeTimetableCompanyKey` (internal)
- `frontend/src/pages/timetable/TimetableContactDrawer.tsx` — `loadKundenDb` + resolver, `dema-kunden-db-changed` listener, CRM snapshot UI; removed header inputs for cust. no. / industry
- `frontend/src/contexts/LanguageContext.tsx` — `timetableContactCrmSnapshotAria`, `timetableContactCrmUnlinked` (de/en/fr); removed `timetableContactCustomerNoPh`, `timetableContactIndustryPh`
- `docs/FEATURE-LOG.md`, this file — FEATURE-175

### What It Does (Technical)
1. On render and when `dema-kunden-db-changed` fires, `loadKundenDb()` is re-read.
2. `resolveKundeForTimetableRow(db, draft.company_name, profile.customer_number)` returns `KundenStamm | null`.
3. UI displays `kunden_nr` and `branche`; if null, shows em dash and localized unlinked hint.

### Data It Accepts / Emits
| Source | Field | Use |
|--------|-------|-----|
| Timetable draft | `company_name`, `contact_profile.customer_number` | Lookup only (no longer written from header) |
| Kunden DB | `kunden_nr`, `firmenname`, `branche` | Display when matched |

### Database
- **Engine:** Browser `localStorage` key `dema-kunden-db` / in-memory state
- **Tables Affected:** N/A (frontend stores)
- **Schema Changes:** None
- **Key Queries:** Linear scan on `kunden` array (small local dataset)

### API Endpoints (if applicable)
None.

### State / Store (if applicable)
- **Store file:** `frontend/src/store/kundenStore.ts`
- **Persisted:** Existing timetable `contact_profile.customer_number` / `industry` may remain in saved JSON but are **not** edited from this header; industry display is **always** from master `branche` when linked.

### i18n Keys Added
- `timetableContactCrmSnapshotAria` — de/en/fr
- ~~`timetableContactCrmUnlinked`~~ — superseded by FEATURE-176 hints + demo keys
