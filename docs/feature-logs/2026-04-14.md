## FEATURE-251: Timetable contact drawer — Bemerkungen as dated chat-style note thread
**Date:** 2026-04-14 
**Author/Agent:** Cursor AI 
**Status:** Added 

### What Was Added
**Correspondence / Bemerkungen** in the purchase timetable contact drawer is no longer a single undated textarea. Notes are stored as a **dated log** (`activity_notes_log`) and shown as **scrollable bubbles** with **locale-aware date and time** inside each box (chat-style). Users **add** notes via a composer and **delete** individual entries. Legacy **`activity_notes`** still loads and appears as **one virtual bubble** timestamped from the row’s **`scheduled_at`** until the user adds a new note (then the legacy text is migrated into the log and cleared). The on-device **smart summary** snippet uses the **latest log entry** (or legacy text). Demo seed row uses a two-entry log for a realistic preview.

### Where It Was Added
- `frontend/src/types/timetable.ts` — `TimetableActivityNoteEntry` and `activity_notes_log` on `TimetableContactProfile` (already present; used end-to-end here).
- `frontend/src/pages/timetable/contactDrawerFormUtils.ts` — `buildActivityNotesDisplayList`, `getActivityNotesLastSnippet`, `getActivityNotesPlainText`, `newActivityNoteId`, `activityNotesLegacyTimestampIso`, virtual legacy id constant.
- `frontend/src/store/timetableStore.ts` — `normalizeActivityNoteEntry`, `normalizeContactProfile` persists `activity_notes_log`; first demo row seed uses `activity_notes_log` instead of long `activity_notes`.
- `frontend/src/pages/timetable/components/TimetableContactDrawerFormBody.tsx` — chat thread UI, composer, add/delete actions.
- `frontend/src/pages/timetable/TimetableContactDrawer.tsx` — smart bullets use `getActivityNotesLastSnippet`.
- `frontend/src/contexts/LanguageContext.tsx` — `timetableContactActivity*` keys in **de, en, fr, es, it, pt, tr, ru, hi, ar, zh, ja**.

### What It Does (Technical)
1. **Load:** `normalizeContactProfile` parses `activity_notes_log` array entries with `id`, `at` (ISO), `text`.
2. **Display:** `buildActivityNotesDisplayList` returns sorted real entries, or a single **virtual** row when only `activity_notes` exists; timestamps formatted with `Intl.DateTimeFormat(localeTag, { dateStyle: 'medium', timeStyle: 'short' })`.
3. **Add:** Appends `{ id, at: now ISO, text }`; if migrating from legacy, prepends a real entry carrying legacy text with `at` from `activityNotesLegacyTimestampIso(scheduled_at)` then appends the new note; clears `activity_notes`.
4. **Delete:** Removes by `id` from `activity_notes_log`, or clears `activity_notes` for the virtual legacy row.
5. **Smart summary:** `getContactSmartBullets` uses `getActivityNotesLastSnippet` for the correspondence line.

### Data It Accepts / Emits
| Field | Type | Required | Description |
|----|---|----|----|
| `activity_notes_log` | `TimetableActivityNoteEntry[]` | No | Dated correspondence; preferred persistence from UI |
| `activity_notes` | `string` | No | Legacy single block; still normalized for old stored rows |
| `TimetableActivityNoteEntry.id` | `string` | Yes | Stable id for delete |
| `TimetableActivityNoteEntry.at` | `string` | Yes | ISO 8601 timestamp |
| `TimetableActivityNoteEntry.text` | `string` | Yes | Note body |

### Database
- **Engine:** Browser `localStorage` (`dema-purchase-timetable-v6`)
- **Tables Affected:** N/A (JSON on `TimetableEntry.contact_profile`)
- **Schema Changes:** New optional `activity_notes_log` on persisted `contact_profile`; legacy `activity_notes` unchanged for compatibility
- **Key Queries:** None

### API Endpoints (if applicable)
None

### State / Store (if applicable)
- **Store file:** `frontend/src/store/timetableStore.ts`
- **Actions/Selectors added:** `normalizeActivityNoteEntry` (internal); `normalizeContactProfile` extended
- **Persisted:** Yes — via existing timetable save path

### i18n Keys Added
`timetableContactActivityAddNote`, `timetableContactActivityComposerPlaceholder`, `timetableContactActivityDeleteNoteAria`, `timetableContactActivityEmpty` — **de, en, fr, es, it, pt, tr, ru, hi, ar, zh, ja**.
