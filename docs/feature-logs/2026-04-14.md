## FEATURE-243: Customer modal header — audit trail as animated status chips
**Date:** 2026-04-14 
**Author/Agent:** Cursor AI 
**Status:** Modified 

### What Was Added
**Created by** and **last edited / last saved** metadata under the address line are no longer plain text; they appear as **`customers-modal-genz-h-chip`** pills in the same horizontal strip as Status, VAT, profile %, risk, and document deadlines, with the same **conic orbit animation** and staggered entrance as other chips. The last-audit pill uses **“Last edited by”** (with pencil icon) when `updated_at !== created_at` and a named editor exists; otherwise **“Last saved”** with save icon. **“Created by”** uses a distinct slate palette (`data-dema-chip="audit-created"`). When no timestamps or creators exist, a muted pill shows **“No edit history available”** (`audit-muted`).

### Where It Was Added
- `frontend/src/components/NewCustomerModal.tsx` — `headerChipsData` extended (`createdAuditLine`, `auditHeaderShowsEditedLabel`, `auditMutedChip`); removed plain-text audit block; chips for `audit-created`, `audit` (icons + dynamic label), `audit-muted`.
- `frontend/src/index.css` — palettes for `[data-dema-chip='audit-created']` and `[data-dema-chip='audit-muted']`; `:nth-child(9)` / `(10)` stagger delays for extra strip children.

### What It Does (Technical)
1. `headerChipsData` builds formatted `createdAuditLine` / `lastEditedLine` with shared `fmtAudit` and locale-aware `toLocaleString`.
2. Edit mode no longer renders the gray flex row above the chip strip; audit info only appears in `customer360-strip-chips`.
3. Chip strip children inherit `.customer360-strip-chips > *` enter + orbit animations unless `prefers-reduced-motion`.

### Data It Accepts / Emits
| Field | Type | Required | Description |
|----|---|----|----|
| `createdAuditLine` | `string \| null` | No | `created_by_name · short datetime` when edit + stamm has creator |
| `lastEditedLine` | `string \| null` | No | Editor (or Unknown) · `updated_at` when edit + timestamp |
| `auditHeaderShowsEditedLabel` | `boolean` | No | Drives label + pencil vs save icon |
| `auditMutedChip` | `boolean` | No | True when edit stamm has neither created nor last-edited lines |

### Database
- **Engine:** N/A (reads existing `KundenStamm` fields on edit)
- **Tables Affected:** N/A
- **Schema Changes:** None
- **Key Queries:** None

- **Persisted:** No

### i18n Keys Added
None — reuses `auditCreatedBy`, `auditLastEditedBy`, `customer360ChipLastEdited`, `auditNoTrail`, `auditUnknownEditor`.

### Dependencies Added

## FEATURE-247: Customer modal — responsive “Beziehungen & Risiko” column title
**Date:** 2026-04-14 
**Author/Agent:** Cursor AI 
**Status:** Modified 

### What Was Added
Column **4** title **“Kundenbeziehungen & Risikoanalyse”** was hard to fit in **2-column** edit layout on viewports **below the 4-column breakpoint** (`min-[1420px]`). Below that width the UI shows **`customerModalColBeziehungenRisikoShort`**; at **`min-[1420px]`+** it shows the full **`customerModalColBeziehungenRisiko`** string. The `<p>` exposes the full label as **`title`** for hover/tooltip. CSS class **`customers-modal-genz-kunde-col-title--beziehungen`** adds **`text-wrap: balance`**, slightly tighter **letter-spacing** and **line-height** under 1420px, and Tailwind **`max-[1419px]:text-[11px]`** for a compact but readable band.

### Where It Was Added
- `frontend/src/components/NewCustomerModal.tsx` — column 4 heading markup.
- `frontend/src/index.css` — `.customers-modal-genz-kunde-col-title--beziehungen` rules.
- `frontend/src/contexts/LanguageContext.tsx` — new key **`customerModalColBeziehungenRisikoShort`** in `MESSAGES.de` / `en` / `fr` and all nine **`LANGUAGE_OVERRIDES`** locales.

### What It Does (Technical)
1. Two `<span>`s toggled with **`inline min-[1420px]:hidden`** / **`hidden min-[1420px]:inline`** (matches `editKundeGridClass` wide layout).
2. i18n: English fallbacks for `t()` remain English per project rules.

### Data It Accepts / Emits
N/A

### Database
N/A

### API Endpoints (if applicable)
None

### State / Store (if applicable)
N/A

### i18n Keys Added
- `customerModalColBeziehungenRisikoShort` — **de** / **en** / **fr** + **es, it, pt, tr, ru, hi, ar, zh, ja** overrides.

### Dependencies Added
None

### Notes / Known Limitations
Breakpoint **1420px** is aligned with the customer **Kunde** grid’s **`min-[1420px]:grid-cols-12`** four-column mode.
