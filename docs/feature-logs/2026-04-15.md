## FEATURE-258: Timetable Bemerkungen — large comfort-reading dialog
**Date:** 2026-04-15 
**Author/Agent:** Cursor AI 
**Status:** Added 

### What Was Added
In the calendar **customer contact** drawer, the **Bemerkungen** (dated activity notes) column now has a **“Large view”** control that opens the same notes thread in a **near full-screen modal** with **larger typography**, **spacious cards**, and **44px-class touch targets** so staff can read long correspondence comfortably (accessibility / older users). The overlay stacks above the drawer (`z-[110]`), supports **backdrop click** and **Escape** to close, and reuses the same `patchDraft` data path as the inline thread.

### Where It Was Added
- `frontend/src/pages/timetable/TimetableContactDrawer.tsx` — `bemerkungenComfortOpen` state; header row with open button; second `createPortal` overlay with dialog chrome; Escape order (overlay before smart summary before drawer close).
- `frontend/src/pages/timetable/components/TimetableActivityNotesThread.tsx` — new `layout="comfortOverlay"` variant: expanded scroll region, `text-base`/`text-lg` note body, larger composer, `min-h-11` icon actions, rounded-xl cards.
- `frontend/src/contexts/LanguageContext.tsx` — i18n keys `timetableContactBemerkungenOpenLarge`, `timetableContactBemerkungenLargeTitle`, `timetableContactBemerkungenLargeHint` in **de / en / fr** and all **LANGUAGE_OVERRIDES** locales (es, it, pt, tr, ru, hi, ar, zh, ja).

### What It Does (Technical)
1. User clicks **Large view** beside the Bemerkungen heading → `setBemerkungenComfortOpen(true)`.
2. A portal renders a `role="dialog"` overlay: title + hint from `t()`, primary **Close** control, backdrop `onClick` closes.
3. `TimetableActivityNotesThread` mounts with `layout="comfortOverlay"` — same `buildActivityNotesDisplayList` / `patchDraft` as the embedded column, so edits stay in the shared `draft` until **Save contact**.
4. `Escape` key closes the overlay first (does not close the whole drawer while overlay is open).
5. Switching timetable row (`entry.id`) resets the overlay closed.

### Data It Accepts / Emits
| Field | Type | Required | Description |
|----|---|----|----|

| `patchDraft` | function | Yes | Updates draft + marks dirty like the inline column |
| `bemerkungenComfortOpen` | `boolean` | Yes | Controls portal visibility |

### State / Store (if applicable)

## FEATURE-267: Purchase timetable follow-up — poll-based desktop reminders
**Date:** 2026-04-15 
**Author/Agent:** Cursor AI 
**Status:** Modified 

### What Was Added
**Desktop follow-up alerts** no longer rely on a single `setTimeout` (unreliable when the tab is in the background). A global **`TimetableFollowUpDueWatcher`** now **polls `loadTimetableDb()` every 5 seconds** (and on **tab visibility**) and calls **`processDueTimetableFollowUpReminders`**, which fires **`Notification`** when `follow_up_at` is due, with **sessionStorage deduplication** per `(row id, follow_up_at)` and a **6-hour lateness cap** to avoid stale spam. **`scheduleTimetableDesktopReminder`** is retained as a no-op for compatibility.

### Where It Was Added
- `frontend/src/pages/timetable/timetableReminderNotify.ts` — polling delivery + notified keys
- `frontend/src/components/TimetableFollowUpDueWatcher.tsx` — interval + `visibilitychange`
- `frontend/src/App.tsx` — mount watcher for authenticated shell
- `frontend/src/pages/timetable/TimetablePage.tsx` — removed timer-based schedule call; toast copy uses updated **`timetableQuickRemindToastNotifOn`**
- `frontend/src/contexts/LanguageContext.tsx` — updated **`timetableQuickRemindToastNotifOn`** (de, en, fr + overrides)

### What It Does (Technical)
1. Every **5s**, watcher loads timetable DB from **localStorage**, scans rows with **`follow_up_at`**, **`!is_completed`**, **`now >= due`**, **`now - due <= 6h`**, and not yet in **`dema-tt-followup-notified-v1`** → **`new Notification`** → persist key.
2. When the user returns to the tab, an **immediate** scan runs.

### Data It Accepts / Emits
| Field | Type | Required | Description |
|----|---|----|----|
| `dema-tt-followup-notified-v1` | `string[]` (sessionStorage) | No | Dedup keys `id|iso`. |

None

### State / Store (if applicable)
- **Store file:** `frontend/src/store/timetableStore.ts` (read-only via `loadTimetableDb`)
- **Actions/Selectors added:** None
- **Persisted:** sessionStorage for notified keys only

### i18n Keys Added
None (updated **`timetableQuickRemindToastNotifOn`** copy only).

## FEATURE-280: Settings Typography — dark mode surface fix
**Date:** 2026-04-15 
**Author/Agent:** Cursor AI 
**Status:** Modified 

### What Was Added
**Display → Typografie** sub-panels no longer use a **light-only gradient** (`from-slate-50/90 to-white`) that **skipped** the global dark-theme surface remaps in `index.css`, which caused **light grey cards** with **light-on-light** slate text and **inverted** chip contrast. 2. Global `text-slate-*` remaps then align with the **dark** surface (readable light grey body copy).
3. `bg-blue-50/60` on the selected typeface tile now receives the same **tinted navy** treatment as `bg-blue-50` / `bg-blue-50/70`.
