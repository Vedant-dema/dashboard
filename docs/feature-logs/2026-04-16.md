## FEATURE-307: Timetable contact drawer — upload documents (toolbar)
**Date:** 2026-04-16 
**Author/Agent:** Cursor AI 
**Status:** Added 

### What Was Added
In the **timetable customer contact drawer** workspace toolbar, a new **pill-style control** (**folder icon** + label) sits **immediately before** the **Kurzfassung** (smart summary) button. It opens a **multi-file picker**; files up to **2 MB** each are stored either in the **CRM customer documents** (`customerRepository.addKundenUnterlage`) when the row **resolves to a master customer**, or on the **timetable row** under **`contact_profile.timetable_unterlagen`** (data URLs) when **no CRM match** exists. Row-local uploads **auto-persist** the finalized timetable entry via existing **`onPersist`**.

### Where It Was Added
- `frontend/src/types/timetable.ts` — **`TimetableContactUnterlage`** type; **`timetable_unterlagen`** on **`TimetableContactProfile`**.
- `frontend/src/pages/timetable/TimetableContactDrawer.tsx` — **`finalizeTimetableEntryForSave`** (shared with **Save**); **`readFileAsDataURL`**; hidden **`<input type="file" multiple>`** + **Upload** button before smart summary; **`handleTimetableUnterlagenFiles`**.
- `frontend/src/contexts/LanguageContext.tsx` — **`timetableContactUploadDocs`**, **`timetableContactUploadDocsAria`** in **de / en / fr** and **9 locale overrides**.

### What It Does (Technical)
1. User clicks **Upload** → **`unterlagenInputRef.click()`** opens the native file dialog.
2. **`onChange`**: if **`resolveKundeForTimetableRow`** found a **`KundenStamm`**, each file is **`addKundenUnterlage`** + **`saveLocalDb`** and **`dema-kunden-db-changed`** is dispatched; else files are appended to **`timetable_unterlagen`** on a cloned entry.
3. Row path calls **`finalizeTimetableEntryForSave`** then **`onPersist`** (clears dirty flag).
4. Oversize / read errors reuse **`customersFileTooBig`** / **`customersFileReadError`** via **`window.alert`**.

### Data It Accepts / Emits
| Field | Type | Required | Description |
|---|---|---|---|
| Files | `FileList` | Yes | User-selected files from picker |
| `timetable_unterlagen[]` | `TimetableContactUnterlage` | No | Persisted on row when no CRM customer match |

### Database
- **Engine:** Browser `localStorage` — **`dema-kunden-db`** (CRM path) and timetable store (row path via parent **`upsertTimetableEntry`**)
- **Tables Affected:** `unterlagen` in **`KundenDbState`** when linked; **`contact_profile.timetable_unterlagen`** on **`TimetableEntry`** otherwise
- **Schema Changes:** New optional **`timetable_unterlagen`** array on **`TimetableContactProfile`**; new interface **`TimetableContactUnterlage`**
- **Key Queries:** None — uses **`customerRepository`** helpers

### API Endpoints (if applicable)
None

### State / Store (if applicable)
- **Store file:** `frontend/src/store/kundenStore.ts` (via **`customerRepository`**) when CRM-linked
- **Actions/Selectors added:** None — uses **`addKundenUnterlage`** / **`saveLocalDb`**
- **Persisted:** Yes — customer **`unterlagen`** or timetable JSON row

### i18n Keys Added
`timetableContactUploadDocs`, `timetableContactUploadDocsAria` (primaries + **es, it, pt, tr, ru, hi, ar, zh, ja** overrides)

### Dependencies Added
None

### Notes / Known Limitations
- Row-level attachments are **data URLs** on the JSON row (same demo pattern as customer uploads); large totals can grow **localStorage**. **2 MB** cap per file matches **Customers** demo.
- Uploaded files are **not listed** in the drawer UI yet — they exist in **customer documents** or raw **`contact_profile`** for future surfacing.
