# Learner Operating Guide

This folder converts your daily work into structured skill growth.

## Naming convention
- Daily file: `YYYY-MM-DD.md`
- Use `templates/daily-learning-template.md` as base.
- Standard cadence: Monday-Friday only.

## Learning principles
- Learn only what supports active backlog items.
- Keep sessions focused: 30 to 90 minutes/day.
- Always produce one applied output (code note, checklist, draft design, or test case).
- Track confidence (1-5) and next skill gap daily.

## Daily workflow
1. Pick one backlog-linked topic.
2. Add up to 3 resources.
3. Execute one practical exercise related to current sprint.
4. Write reflection and next-step.
5. Weekend learning is optional catch-up, not required.

## Anti-patterns
- Consuming tutorials without building.
- Learning topics unrelated to current milestones.
- Long study sessions without applied output.
